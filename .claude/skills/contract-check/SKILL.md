# 前后端契约一致性检查

## 触发场景

- 新增或调整后端接口、请求参数、响应字段、枚举、错误码或鉴权语义。
- 前端新增 API 调用、页面依赖新字段、Mock 或联调状态变化。
- Review 或上线前需要确认前后端契约是否一致。

## 核心原则

- OpenAPI `/v3/api-docs` 是接口契约唯一真源。
- 接口联调状态只描述能否 mock、可联调或已废弃，不替代字段契约。
- 接口状态合法值和状态清单字段以 `.agents/references/interface-status-model.md` 为准。
- 契约检查只输出一致性结论，不负责把接口状态提升为 `ready`。
- 没有真实 OpenAPI 或没有读取前端调用时，结论必须标注“未验证”。
- 没有确认接口状态时，必须标注“状态未确认”。

## 工作流

1. 获取 OpenAPI：优先使用任务提供的 `/v3/api-docs` JSON 或可访问地址。
2. 提取后端契约：路径、HTTP 方法、请求参数、响应字段、枚举、错误码、鉴权语义。
3. 提取前端消费：前端 API、相关页面、store、Mock 和错误处理分支。
4. 确认接口状态：记录状态值、确认人、更新时间和前端动作；无法确认时标注“状态未确认”。
5. 对照检查：路径/方法、参数名与类型、响应字段、枚举值、分页语义、错误码、Mock 字段。
6. 输出不一致清单：包含严重级别、位置、影响和建议修复。

## 严重级别

- `P0`：接口不可用、字段完全不匹配或核心流程阻断。
- `P1`：关键字段、错误码、鉴权或分页语义不一致。
- `P2`：边界字段、Mock、失败路径或类型约束不一致。
- `P3`：文档、命名、注释或低风险优化。

## 输出要求

- 联动交付需包含契约来源、接口状态、后端字段/接口到前端类型/调用/页面行为的映射、受影响模块和未验证项。

```markdown
## 契约一致性检查报告

- OpenAPI 来源：
- 前端检查范围：
- 接口状态：<状态值或状态未确认>
- 状态确认人：
- 状态更新时间：
- 结论：通过 / 不通过 / 未验证

| 级别 | 后端契约 | 前端位置 | 问题 | 建议 |
|------|----------|----------|------|------|
```
