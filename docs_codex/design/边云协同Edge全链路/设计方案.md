## 3. 方案

### 3.1 总体方案

```text
Edge Node
  -> EMQX Broker
    -> HTTP Authentication
    -> Rule Engine / HTTP Sink
      -> Cloud Edge Runtime Ingest
        -> Runtime Event
        -> Runtime Snapshot
        -> Metric Minute
        -> Monitor Alert
        -> Task State Machine
  <- EMQX Management API publish

Cloud Admin UI
  -> /admin-api/edge/**
```

架构决策：

1. Cloud 不作为长期 MQTT 订阅消费者。
2. 边缘节点直连 EMQX，EMQX 通过 HTTP Authentication 调 Cloud 鉴权。
3. 上行事件由 EMQX Rule Engine / HTTP Sink 转发到 `POST /admin-api/edge/runtime/ingest`。
4. 下行命令由 Cloud 调 EMQX Management API `/api/v5/publish` 发布。
5. 管理端以 REST API 与定时刷新为 V1 主方案。

### 3.5 关键流程

#### 节点认证与注册

1. 前端创建节点，后端生成 `deviceId`、凭证、`topicRoot`。
2. 边缘端使用 `clientId=edge-node:{deviceId}`、`username={deviceId}`、`password=secret` 连接 EMQX。
3. EMQX HTTP Authentication 调 `POST /admin-api/edge/runtime/authenticate`。
4. `EdgeRuntimeServiceImpl#authenticate` 校验节点启用、凭证 ACTIVE、密码匹配，并返回 per-device ACL。
5. 边缘端发布 `{topicRoot}/lifecycle/register`。
6. `POST /admin-api/edge/runtime/ingest` 处理 `register`，首次绑定 `hardwareFingerprint`，推进激活与快照状态。

#### 心跳与监控

1. 边缘端发布 `{topicRoot}/telemetry/heartbeat`。
2. `EdgeRuntimeServiceImpl#ingest` 落原始事件并更新 snapshot、metric、alert。
3. `EdgeRuntimeServiceImpl#scanHeartbeatTimeouts` 按 `vmesh.edge.runtime.heartbeat-scan-interval-ms` 巡检超时节点。
4. 监控页通过 `/edge/monitor/overview`、`/edge/monitor/heartbeat/page`、`/edge/monitor/device-abnormal/page`、`/edge/monitor/system-status/page` 展示状态。

#### 任务下发

1. 前端调用 `POST /edge/task/dispatch`，请求体为 `EdgeTaskDispatchReqVO`。
2. `EdgeTaskServiceImpl` 生成 `taskKey`、`commandKey`、`commandTopic` 和命令 envelope。
3. Cloud 调 `{vmesh.edge.runtime.emqx-api-base-url}/publish`，向 `{topicRoot}/commands/{commandKey}` 发布命令。
4. 边缘端执行后发布 `{topicRoot}/events/task-progress`。
5. 任务中心根据 `task_progress` 更新任务状态和日志。
