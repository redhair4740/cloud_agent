# 边缘监控与任务流水设计方案

## 1. 背景

- 需求来源：对齐 `moon_cloud_design/src/views/EdgeHealthMonitor.vue` 与 `moon_cloud_design/src/views/EdgeTaskRecord.vue`。
- 当前现状：生产前端已存在 `moon_cloud_frontend/src/views/edge/monitor/index.vue`、`moon_cloud_frontend/src/views/edge/task/index.vue`、`TaskDetailDrawer.vue`；后端已存在 `EdgeMonitorController` 和 `EdgeTaskController`。
- 触发原因：旧设计含注册链路跟踪等历史视图，最新页面目标收敛为心跳监控、设备异常、系统状态预警、任务流水。

## 2. 目标

| 编号 | 目标 | 验收指标 |
|------|------|----------|
| G-1 | 监控页按最新原型收敛 | 展示心跳、设备异常、系统状态预警，不把注册链路作为主视图 |
| G-2 | 设备异常只展示异常 | 正常设备不进入异常列表 |
| G-3 | 系统状态预警字段完整 | CPU、NPU/GPU、内存、存储、温度、丢包率、建议可展示 |
| G-4 | 任务流水契约化 | 统计、筛选、分页、日志、撤回、重试接口明确 |

## 3. 页面与接口

### 3.1 边缘监控页

| 区域 | 接口 | 源码证据 |
|------|------|----------|
| 顶部统计卡 | `GET /edge/monitor/overview` | `EdgeMonitorController#getOverview`、`EdgeMonitorOverviewRespVO` |
| 心跳监控 | `GET /edge/monitor/heartbeat/page` | `EdgeMonitorController#getHeartbeatPage`、`EdgeMonitorHeartbeatRespVO` |
| 设备异常 | `GET /edge/monitor/device-abnormal/page` | `EdgeMonitorController#getDeviceAbnormalPage`、`EdgeMonitorAbnormalDeviceRespVO` |
| 系统状态预警 | `GET /edge/monitor/system-status/page` | `EdgeMonitorController#getSystemStatusPage`、`EdgeMonitorSystemStatusRespVO` |
| 节点详情 | `GET /edge/monitor/node/{nodeId}/detail` | `EdgeMonitorController#getNodeDetail` |
| 节点趋势 | `GET /edge/monitor/node/{nodeId}/trend` | `EdgeMonitorController#getNodeTrend` |

边界：

- `/edge/monitor/registration/page` 可保留给审计或排障，但不作为最新监控页主视图。
- `/edge/monitor/high-load/page` 可兼容旧高负载视图，最新主视图优先使用 `/system-status/page`。
- 设备异常枚举保留 `OFFLINE`、`STREAM_LOST`、`FRAME_DROP`、`AUTH_FAILED`。

### 3.2 边缘任务流水页

| 区域 | 接口 | 源码证据 |
|------|------|----------|
| 今日统计 | `GET /edge/task/statistics` | `EdgeTaskController#getStatistics`、`EdgeTaskStatisticsRespVO` |
| 任务分页 | `GET /edge/task/page` | `EdgeTaskController#getTaskPage`、`EdgeTaskRespVO` |
| 任务详情 | `GET /edge/task/{taskKey}` | `EdgeTaskController#getTaskDetail`、`EdgeTaskDetailRespVO` |
| 日志抽屉 | `GET /edge/task/{taskKey}/logs` | `EdgeTaskController#getTaskLogs`、`EdgeTaskLogRespVO` |
| 新任务下发 | `POST /edge/task/dispatch` | `EdgeTaskController#dispatch`、`EdgeTaskDispatchReqVO` |
| 撤回 | `POST /edge/task/{taskKey}/cancel` | `EdgeTaskController#cancelTask` |
| 重试 | `POST /edge/task/{taskKey}/retry` | `EdgeTaskController#retryTask` |

任务类型：

| 枚举 | 含义 | commandKey |
|------|------|------------|
| `OTA` | OTA 底座升级 | `ota_update` |
| `ALGORITHM` | 算法下发 | `algorithm_update` |
| `CONFIG` | 配置下发 | `config_update` |
| `REBOOT` | 设备重启 | `reboot` |
| 其他 | 自定义任务 | `custom_command` |

任务状态建议：

| 状态 | 含义 |
|------|------|
| `CREATED` | 已创建 |
| `QUEUED` | 已排队 |
| `DISPATCHED` | 已下发 |
| `ACKNOWLEDGED` | 边缘端已确认 |
| `PROCESSING` | 执行中 |
| `SUCCESS` | 成功 |
| `FAILED` | 失败 |
| `CANCELED` | 已取消 |
| `TIMEOUT` | 超时 |

## 4. 关键流程

### 4.1 心跳监控

1. 边缘端发布 `{topicRoot}/telemetry/heartbeat`。
2. 后端写入运行态快照和心跳指标。
3. 前端调用 `GET /edge/monitor/heartbeat/page`。
4. 心跳槽位由后端根据 `lastHeartbeatAt` 与 `heartbeatIntervalSec` 生成。
5. 超时节点由巡检任务降级为离线，并打开 `OFFLINE` 告警。

### 4.2 设备异常

1. 边缘端发布 `{topicRoot}/telemetry/device-report`。
2. 后端从设备运行态、告警和最近上报信息中归一异常。
3. 前端调用 `GET /edge/monitor/device-abnormal/page`，只展示异常设备。
4. 异常恢复后设备从异常列表消失，历史进入告警或运行事件查询。

### 4.3 任务流水

1. 前端进入页面时加载统计和第一页任务。
2. 筛选任务类型、状态、关键词时重置页码。
3. 点击日志打开抽屉，加载详情与日志。
4. 失败或超时任务允许重试，未完成任务允许撤回。

## 5. 影响范围

- 后端：`EdgeMonitorController`、`EdgeMonitorServiceImpl`、`EdgeTaskController`、`EdgeTaskServiceImpl`。
- 前端：`moon_cloud_frontend/src/views/edge/monitor/index.vue`、`src/views/edge/task/index.vue`、`src/views/edge/task/components/TaskDetailDrawer.vue`。
- API SDK：`moon_cloud_frontend/src/api/edge/monitor.ts`、`src/api/edge/task.ts`。

## 6. 风险与回滚

- 风险：最新原型与真实 OpenAPI 未做本轮实时比对，字段仍需联调确认。
- 风险：任务类型前端原型曾使用小写值，生产接口建议使用大写枚举，落地时要做映射。
- 回滚方案：保留旧接口兼容，页面可暂时退回当前已有字段集。
- 未验证项：未读取实时 `/v3/api-docs`，未执行前端构建、后端编译或接口联调。
