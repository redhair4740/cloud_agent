# 边云监控与告警控制中心 — 原型图产出计划

## 目标

基于设计方案和 API 文档，为 5 个核心页面产出可交互的 HTML 原型图，沿用项目现有 Element Plus + Vue 3 技术栈和页面布局模式。

## 技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 原型形式 | 独立 HTML 文件（内联 Vue 3 + Element Plus CDN） | 无需构建即可浏览器预览，方便评审 |
| UI 库 | Element Plus 2.x CDN | 与项目实际使用一致，组件行为真实 |
| 图表 | ECharts 5.x CDN | 与项目实际使用一致 |
| 样式 | 内联 SCSS 编译后 CSS + Element Plus 主题 | 还原项目统计卡、表格、抽屉等视觉风格 |
| 交互 | Vue 3 Composition API | 与项目代码风格一致，后续可直接迁移为 .vue 组件 |

## 页面设计细节

### 1. 运行总览

**布局**：三段式（统计卡 → 双列区块 → 全宽区块）

```
┌──────────────────────────────────────────────────────────┐
│ 统计卡片区（6 列响应式）                                    │
│  在线节点 | 在线设备 | 异常设备 | 活跃告警 | 失败任务 | 控制失败  │
├──────────────────────────┬───────────────────────────────┤
│ 重点异常设备（el-table）   │ 告警趋势图（ECharts 折线图）    │
├──────────────────────────┴───────────────────────────────┤
│ 异常节点/设备列表（el-table，含快捷操作）                     │
└──────────────────────────────────────────────────────────┘
```

**统计卡字段**：
- `onlineNodeCount` / `onlineDeviceCount` / `abnormalDeviceCount` / `activeAlertCount` / `failedTaskCount` / `controlFailedCount`

**快捷入口**：查看实时告警、查看控制记录、查看节点详情、查看设备详情

**抽屉**：节点详情抽屉、设备详情抽屉（复用现有 NodeDetailDrawer 模式）

### 2. 实时告警

**布局**：统计条 + 筛选区 + 表格 + 抽屉/弹窗

```
┌──────────────────────────────────────────────────────────┐
│ 统计条：OPEN 数 | ACKED 数 | SILENCED 数 | 全部              │
├──────────────────────────────────────────────────────────┤
│ 筛选区：来源类型 | 严重级别 | 告警类型 | 状态 | 节点 | 关键词   │
├──────────────────────────────────────────────────────────┤
│ 实时告警表格（el-table + Pagination）                       │
│  列：告警类型 | 来源 | 级别 | 状态 | 节点 | 设备 | 摘要 |      │
│      首次触发 | 最近触发 | 操作                              │
│  操作：详情 | 确认 | 静默 | 关闭 | 一键控制                   │
└──────────────────────────────────────────────────────────┘

抽屉：告警详情（含基本信息 + 关联策略 + 推荐动作）
弹窗：一键控制弹窗（选择控制类型 → 确认下发）
```

**筛选字段**：sourceType / severity / alertType / status / nodeId / deviceId / keyword

**表格列**：alertKey / sourceType / alertType / severity / status / nodeName / deviceName / summary / firstTriggeredAt / lastTriggeredAt + 操作列

**状态标签颜色**：
- OPEN → danger
- ACKED → warning
- SILENCED → info
- RECOVERED → success

**严重级别标签颜色**：
- CRITICAL → danger
- WARN → warning
- INFO → info

### 3. 告警历史

**布局**：筛选区 + 表格 + 时间线抽屉

```
┌──────────────────────────────────────────────────────────┐
│ 筛选区：来源 | 级别 | 类型 | 节点 | 设备 | 策略 | 状态 |      │
│        时间范围 | 关键词                                    │
├──────────────────────────────────────────────────────────┤
│ 告警历史表格（el-table + Pagination）                       │
│  列：告警类型 | 来源 | 级别 | 终态 | 节点 | 设备 | 摘要 |      │
│      首次触发 | 恢复时间 | 操作                              │
│  操作：时间线 | 控制结果                                     │
└──────────────────────────────────────────────────────────┘

抽屉 1：生命周期时间线（el-timeline）
  TRIGGERED → RETRIGGERED → ACKED → CONTROL_TRIGGERED →
  CONTROL_SUCCEEDED/FAILED → RECOVERED → CLOSED

抽屉 2：控制结果详情
```

**时间线事件类型与图标**：
- TRIGGERED → Warning 图标 + 橙色
- RETRIGGERED → Warning 图标 + 橙色
- ACKED → User 图标 + 蓝色
- RECOVERED → Success 图标 + 绿色
- CLOSED → CircleCheck 图标 + 灰色
- CONTROL_TRIGGERED → Promotion 图标 + 紫色
- CONTROL_SUCCEEDED → SuccessFilled 图标 + 绿色
- CONTROL_FAILED → CircleClose 图标 + 红色

### 4. 告警策略

**布局**：筛选区 + 表格 + 编辑抽屉

```
┌──────────────────────────────────────────────────────────┐
│ 筛选区：告警类型 | 作用范围 | 动作类型 | 启停状态 | 关键词      │
│  操作按钮：新增策略                                          │
├──────────────────────────────────────────────────────────┤
│ 策略表格（el-table + Pagination）                           │
│  列：策略名称 | 作用范围 | 告警类型 | 级别 | 动作 | 优先级 |    │
│      状态 | 操作                                            │
│  操作：编辑 | 启停 | 复制 | 删除                              │
└──────────────────────────────────────────────────────────┘

抽屉：策略编辑（el-form + el-steps 或分区表单）
  区块 1：基本信息（名称、作用范围、告警类型、级别）
  区块 2：触发条件（指标、运算符、阈值、持续时长/连续次数）
  区块 3：恢复条件（可选）
  区块 4：动作配置（动作类型、通知渠道、控制类型、任务映射）
  区块 5：优先级与备注
```

**策略编辑抽屉分区**：
- 基本信息：name / scopeType / scopeIds（级联选择） / alertType / severity
- 触发条件：triggerRule（metric / operator / threshold / durationSeconds / consecutiveTimes / timeWindowMinutes）
- 恢复条件：recoverRule（同触发条件结构，可选填）
- 动作配置：actionType / actionConfig（notifyChannels / controlType / taskType / resourceId / inlinePayload）
- 其他：priority / enabled / remark

### 5. 硬件控制

**布局**：筛选区 + 快捷控制区 + 控制记录表 + 弹窗/抽屉

```
┌──────────────────────────────────────────────────────────┐
│ 筛选区：控制类型 | 来源 | 状态 | 节点 | 时间范围              │
│  快捷操作：发起控制                                          │
├──────────────────────────────────────────────────────────┤
│ 快捷控制区（el-card，3~4 个常用控制快捷按钮）                  │
│  重启节点 | 重启服务 | 切换模式 | 刷新配置                     │
├──────────────────────────────────────────────────────────┤
│ 控制记录表格（el-table + Pagination）                        │
│  列：控制类型 | 来源 | 节点 | 设备 | 关联告警 | 状态 | 结果 |   │
│      创建时间 | 完成时间 | 操作                               │
│  操作：重试 | 查看任务日志                                    │
└──────────────────────────────────────────────────────────┘

弹窗：控制下发（选择节点 → 选择控制类型 → 填写参数 → 确认）
抽屉：任务日志（复用 TaskDetailDrawer 模式，展示执行步骤和结果）
```

**控制下发弹窗字段**：nodeId / deviceId / controlType / sourceType / alertId / taskType / resourceId / inlinePayload / reason

**控制状态标签**：
- PENDING → info
- DISPATCHED → primary
- EXECUTING → warning
- SUCCEEDED → success
- FAILED → danger
- CANCELLED → info

## 约束与边界

1. 原型为纯前端静态文件，不连接真实后端 API
2. 所有数据为 Mock 数据，交互为前端模拟（如确认操作后状态变化）
3. 不做真实权限控制（v-hasPermi 仅做展示）
4. ECharts 图表使用静态数据，不做实时刷新模拟
5. 原型文件不纳入项目构建流程，仅作为评审和参考用途
6. 后续实现阶段可直接参考原型中的布局结构和组件用法，迁移为 .vue 组件
