### 1.1 命名规范

**统一命名格式**：`hh-mm-ss-类型标签-文档名称.md`

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| hh-mm-ss | 文档创建时间，精确到秒，**放在最前面保证默认排序按时间** | `14-30-05` |
| 类型标签 | 文档类型，固定枚举值 | `设计方案`、`交接文档`、`治理评审`、`协作分析`、`部署方案` |
| 文档名称 | 文档内容概述 | `边云协同Edge全链路生产级设计` |

**关联规则**：同一时间点（或相近时间点）的设计方案和交接文档，通过 `hh-mm-ss` 建立关联。

示例：
```
2026-04-22/
├── 14-30-00-设计方案-边云协同Edge全链路生产级设计.md    ← 14:30 创建的设计方案
└── 23-10-00-交接文档-Edge全链路落地任务交接.md          ← 23:10 创建的交接文档
```

文件按名称排序时，自然按时间先后排列，同一时间段内的方案和交接文档紧邻显示。

### 1.2 类型标签枚举

| 类型标签 | 适用目录 | 说明 |
|---------|---------|------|
| `设计方案` | design/ | 技术方案、架构设计、实施路线图 |
| `交接文档` | design/ | AI 续接文档，记录进度与下一步 |
| `治理评审` | governance/ | 规范评估、治理优化建议 |
| `协作分析` | governance/ | 协作模式分析、项目分析 |
| `部署方案` | deployment/ | 部署方案、运维方案、发布最佳实践 |

### 1.3 与 review-tracking 命名对比

| 目录 | 命名格式 | 说明 |
|------|---------|------|
| review-tracking/ | `review清单-hh-mm-ss.md` | 已有格式，保持不变 |
| design/ | `hh-mm-ss-设计方案-名称.md` / `hh-mm-ss-交接文档-名称.md` | 新格式 |
| governance/ | `hh-mm-ss-治理评审-名称.md` / `hh-mm-ss-协作分析-名称.md` | 新格式 |
| deployment/ | `hh-mm-ss-部署方案-名称.md` | 新格式 |

命名风格统一：`hh-mm-ss-类型标签[-名称].md`
