# 前后端契约一致性检查

## 概览

本 skill 用于联动任务收尾或契约审计。OpenAPI `/v3/api-docs` 是唯一契约真源，`api-status.yml` 只描述联调状态。

## 工作流

1. 获取 OpenAPI：优先使用任务提供的 `/v3/api-docs` JSON 或可访问地址。
2. 提取后端契约：路径、HTTP 方法、请求参数、响应字段、枚举、错误码、鉴权语义。
3. 提取前端消费：`<frontend-dir>/src/api/**`、相关页面、store、Mock 和错误处理分支。
4. 对照检查：路径/方法、参数名与类型、响应字段、枚举值、分页语义、错误码、Mock 字段。
5. 输出不一致清单：包含严重级别、位置、影响和建议修复。

## 严重级别

- `P0`：接口不可用、字段完全不匹配或核心流程阻断。
- `P1`：关键字段、错误码、鉴权或分页语义不一致。
- `P2`：边界字段、Mock、失败路径或类型约束不一致。
- `P3`：文档、命名、注释或低风险优化。

## 输出要求

```markdown
## 契约一致性检查报告

- OpenAPI 来源：
- 前端检查范围：
- api-status 状态：
- 结论：通过 / 不通过 / 未验证

| 级别 | 后端契约 | 前端位置 | 问题 | 建议 |
|------|----------|----------|------|------|
```

未获取真实 OpenAPI 或未读取前端调用时，必须标注“未验证”。
