# 后端对象分层规则

## 默认分层

- Controller 只处理协议层输入输出、鉴权注解和参数校验，不直接访问 Mapper。
- Service 负责业务编排、事务边界和跨对象协作。
- Mapper 只负责数据库访问，不承载业务判断。
- DO 对应数据库结构，VO/DTO 对应接口或应用层传输结构。

## 目录组织

- 默认采用轻量模块化，不为单一简单流程提前引入 BO、Assembler、Facade 等额外层。
- 只有复杂聚合、跨子域编排或真实重复转换出现时，才新增中间层。
- 测试目录优先镜像生产目录，不额外制造平行体系。

## 禁止事项

- 禁止 Controller 直接调用 Mapper。
- 禁止 Controller 承载复杂业务编排。
- 禁止跨域直接复用别的域 DO/Mapper 作为常规实现手段。
- 禁止为了“看起来分层完整”提前制造空抽象。
