# 仓库通用基线

## 沟通与证据

- 全程使用简体中文沟通、分析、注释与交付说明。
- 关键判断必须证据化：优先给出文件路径、方法名、配置键、命令、参数和真实行为。
- 结论需可复查，不用“应该”“可能”替代可执行动作。

## 注释基线

- 注释必须使用简体中文。
- 注释解释意图、边界、取舍、失败路径与风险，不复述显而易见的代码字面含义。
- 复杂流程的详细注释位置和交付口径按对应后端、前端或文档 skill 执行。

## 验证口径

- 未执行真实验证时，必须明确标注“未验证”或“未执行测试”。
- 只有存在实际验证证据时，才可使用“已修复”“已通过”“已完成”。
- 禁止把静态分析、阅读代码、局部推断表述成已验证结论。
- 声称编译通过时，必须同时给出真实编译命令、执行范围与结果。

## 协作与联动

- 只要需求涉及前后端契约、字段、错误语义、鉴权语义或联调验收，必须按联动任务处理。
- 禁止单边先改后通知另一侧兜底；跨端变更必须先对齐范围、验收口径和回滚思路。
- 跨端未达成一致前，不做不可逆实现决策。

## 风险边界

- 未经明确确认，不执行高风险操作：批量删除、批量重命名、回滚、外网真实服务调用、生产环境数据写操作。
- 禁止危险 Git 操作作为默认手段，例如硬重置、覆盖式回退、强推覆盖历史。
- 发现敏感信息时仅允许定位与脱敏说明，禁止在对话或文档中回显明文。
