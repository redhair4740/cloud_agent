# 文档模板使用流程

## 触发场景

- 用户要求生成设计方案、协作分析、治理评审、部署方案或交接文档。
- 用户要求记录 Bug 修复、重构计划、评审清单或 review 跟踪项。
- 已完成实现，需要补充可归档的说明、验收标准、风险和未验证项。

## 模板选择

| 场景 | 模板 |
|------|------|
| 通用设计方案或联动方案 | `templates/design-doc.md` |
| Bug 修复工单 | `templates/bug-fix.md` |
| 重构工单 | `templates/refactor.md` |
| 交接文档 | `templates/handover-doc.md` |
| Review 跟踪清单 | `templates/review-checklist.md` |

需要示例时读取 `examples/` 下同名 `.example.md` 文件；不需要示例时不要额外加载。

## 使用流程

1. 先确认文档类型、目标读者、关联任务和是否涉及接口契约。
2. 按文档管理规则确定目录、日期子目录、文件名和 review 状态后缀；新建文档必须先读取当前本地时间并使用实际 `hh-mm-ss`，不得默认使用 `00-00-00` 占位。
3. 读取对应模板，只填充与任务相关的章节，不制造无法验证的内容。
4. 涉及接口契约时写清 OpenAPI 来源、接口状态、确认人和未确认项。
5. 交付前检查“已验证/未验证/风险/回滚/下一步”是否完整。
6. 新增或移动归档文档后，提醒同步更新 `docs/README.md` 索引。

## 输出要求

- 文档正文必须保留真实证据：文件路径、接口路径、命令、配置键或行为说明。
- 未执行验证时必须写“未验证/未执行测试/未编译验证”。
- Review 清单状态后缀只能使用 `已完成` 或 `未完成`。
- 不把模板示例里的业务名、路径或状态当成当前任务事实。
