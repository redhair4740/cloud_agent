## 1. 背景

**需求来源**：产品需求 PRD-2026-0424，用户反馈列表页加载缓慢。

**当前现状**：列表接口返回全量字段，单次响应 2000+ 条记录，前端渲染耗时 >3s。

**触发原因**：性能优化专项。

## 2. 目标

| 编号 | 目标 | 验收指标 |
|------|------|----------|
| G-1  | 列表接口响应时间降至 500ms 以内 | P95 < 500ms |
| G-2  | 前端首屏渲染时间降至 1s 以内 | LCP < 1s |

## 3. 方案

### 3.1 总体方案

后端接口增加分页参数，前端改用分页加载 + 虚拟滚动。

### 3.2 接口契约

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 是 | 页码，从 1 开始 |
| pageSize | int | 是 | 每页条数，最大 100 |

| 无变更 | - | - |

### 3.4 关键流程

1. 前端传入 pageNum、pageSize 请求列表接口
2. 后端 MyBatis-Plus 分页查询，返回 total + records
3. 前端渲染当前页数据，滚动到底部自动加载下一页
