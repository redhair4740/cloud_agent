# 评审清单模板

> 用途：记录代码评审、设计评审或联调评审的检查项、问题与修复建议，供评审跟踪使用。
> 输出目录：`docs/review-tracking/yyyy-MM-dd/review清单-hh-mm-ss-文档名称-状态.md`
> 状态后缀仅允许：`已完成`、`未完成`

## 2. 评审项 <!-- 必填 -->

### 2.1 功能正确性

- [ ] 核心业务逻辑与需求一致
- [ ] 边界条件与异常场景已覆盖
- [ ] 接口契约与设计文档一致（联动任务必检）

### 2.2 代码质量

- [ ] 命名规范符合项目规则（`00-repo-baseline.md` + 对应端规则）
- [ ] 无硬编码魔法值
- [ ] 无冗余/重复代码
- [ ] 对象分层符合 `11-backend-object-layering-rules.md`（后端必检）

### 2.3 安全与风险

- [ ] 无敏感信息硬编码或明文输出
- [ ] 输入校验与权限检查完整
- [ ] 无 SQL 注入 / XSS 等安全风险

### 2.4 前后端联动（仅联动任务）

- [ ] 接口字段、类型、必填与设计文档一致
- [ ] 错误码与错误语义前后端对齐
- [ ] 联调验收口径已确认

### 2.5 测试覆盖

- [ ] 核心路径有对应测试用例
- [ ] 异常路径有对应测试用例
- [ ] 测试结果已验证（非仅静态分析）

- [ ] 不通过：存在严重问题，需修复后重新评审
