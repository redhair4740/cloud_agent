# 跨模型能力适配层

本文件定义 AI_Vision 项目在不同 AI 模型与工具环境下的能力判断、上下文加载与降级策略。模型池会持续变化，本规范不维护具体模型厂商、系列或型号清单，只按实际能力表现选择适配方式。

## 1. 基本原则

- 不按模型品牌或型号预设能力，按当前会话的真实表现与工具环境判断。
- 默认采用“摘要入口 + 按任务类型加载详细规则”的方式：先读 `AGENTS.md` / `CLAUDE.md` / `GEMINI.md` / `.trae/rules/project_rules.md`，再按任务加载 `.ai/rules/*.md`。
- 大部分实际使用模型具备较大上下文，通常可胜任当前 `.ai/` 规范；只有出现上下文不足、漏读规则、工具调用不稳定或输出漂移时才降级。
- 所有降级只减少输入范围或简化表达方式，不改变规则语义与优先级。

## 2. 能力判断维度

| 维度 | 等级 | 判断标准 | 默认策略 |
|------|------|----------|----------|
| 上下文容量 | 充足 | 能稳定处理入口摘要、任务规则、关键代码与验证输出 | 正常按任务加载规则，无需预裁剪 |
| 上下文容量 | 受限 | 读取多份规则后开始遗忘约束、遗漏路径或混淆任务边界 | 只保留核心红线、任务矩阵和当前任务必读规则 |
| 工具能力 | 可直接调用 | 能稳定读取文件、搜索代码、执行命令并返回结构化结果 | 直接执行低风险工具调用，高风险操作按规则确认 |
| 工具能力 | 需人工确认 | 能提出工具调用意图，但参数、权限或结果稳定性不足 | 先输出拟执行命令/路径/影响，再等待确认或人工执行 |
| 工具能力 | 不可调用 | 当前环境无法调用工具或工具结果不可用 | 输出操作清单，由人工执行后回填结果 |
| 指令遵循 | 稳定 | 能持续遵守任务矩阵、业务字典、接口状态和风险边界 | 使用标准 Markdown 规则表达 |
| 指令遵循 | 一般 | 偶发漏读规则或忽略未验证口径 | 重申核心红线，压缩上下文到当前任务相关规则 |
| 指令遵循 | 易漂移 | 多次违反关键约束或混淆业务词/接口状态 | 降级到编号清单，只保留必须/禁止项并要求逐项确认 |
| 文件读取 | 可按路径读取 | 能按路径读取 `.ai/rules/*.md`、`.ai/api-status.yml`、代码文件 | 使用路径引用和按需加载 |
| 文件读取 | 只能读取入口摘要 | 无法主动读取额外文件或只消费入口文件 | 在入口摘要中保留核心红线，必要时让人工粘贴对应规则 |

## 3. 常规加载策略

### 3.1 默认入口

不同工具优先读取自己的入口文件：

- Codex / 通用 Agent：`AGENTS.md`
- Claude Code：`CLAUDE.md`
- Gemini CLI：`GEMINI.md`
- Trae-CN：`.trae/rules/project_rules.md` 与 `.trae/rules/*.md`

入口文件只承载核心红线、任务类型加载矩阵和详细规则路径。详细规则以 `.ai/` 为真实来源。

### 3.2 任务类型加载矩阵

| 任务类型 | 必读规则/文件 |
|----------|---------------|
| 通用任务 | `00-repo-baseline.md` + `01-business-dictionary.md` |
| 后端任务 | 通用规则 + `10-backend-development-rules.md` + `11-backend-object-layering-rules.md` |
| 前端任务 | 通用规则 + `20-frontend-development-rules.md` |
| 前后端联动任务 | 通用规则 + `10-backend-development-rules.md` + `20-frontend-development-rules.md` + `30-fullstack-linkage-rules.md` + `.ai/api-status.yml` |
| Review 任务 | 按被评审对象加载对应规则；联动评审额外加载 `30-fullstack-linkage-rules.md` 与 `.ai/api-status.yml` |

### 3.3 默认不加载内容

- 不默认加载所有 `agents/`，只在任务需要角色职责时读取。
- 不默认加载所有 `skills/`，只在任务语义命中某个 skill 时读取。
- 不默认加载所有模板文件，只有生成对应文档时读取。

## 4. 降级策略

### 4.1 上下文降级

当模型出现遗忘约束、混淆任务边界或无法处理完整上下文时，按以下顺序裁剪：

1. 保留入口摘要、核心红线和任务矩阵。
2. 保留当前任务必读规则，移除无关端规则。
3. 保留当前任务涉及的代码文件、配置文件和验证输出。
4. skills、templates、历史文档只保留被当前任务直接引用的内容。
5. 若仍超限，将当前任务必读规则压缩为“必须/禁止/交付要求”清单。

### 4.2 指令表达降级

默认使用 Markdown 标题、表格和列表。若模型对复杂结构理解不稳定，依次降级：

```text
Markdown 结构 → 编号清单 → 纯文本必须/禁止项
```

降级后仍必须保留：业务字典、OpenAPI 契约真源、`.ai/api-status.yml` 状态真源、Mock 落点、高风险确认、未验证口径。

### 4.3 工具调用降级

| 情况 | 处理方式 |
|------|----------|
| 工具可用且低风险 | 直接读取、搜索、静态检查或运行允许的验证命令 |
| 工具可用但操作高风险 | 先说明操作类型、影响范围、风险，再等待明确确认 |
| 工具参数不稳定 | 先输出拟执行命令、工作目录和预期结果，确认后执行 |
| 工具不可用 | 输出人工操作清单，等待人工回填结果后继续 |

## 5. 运行时自检

每轮任务开始时，AI 应快速自检：

1. 当前任务属于通用、后端、前端、联动还是 Review。
2. 是否已读取入口摘要与当前任务必读规则。
3. 是否需要读取 `.ai/api-status.yml` 或某个 skill。
4. 当前模型是否出现上下文遗忘、规则漏读、工具失败或输出漂移。
5. 若出现问题，按第 4 节降级，并在交付说明中标注“已降级原因”。

## 6. 交付记录要求

发生跨模型适配或降级时，交付说明应包含：

- 实际加载的规则文件与路径。
- 是否发生上下文、指令表达或工具调用降级。
- 降级原因与保留/裁剪内容。
- 未验证范围与后续建议。

未发生降级时，无需额外记录模型型号，只需按项目交付规范说明验证结果。
