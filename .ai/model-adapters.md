# 跨模型能力适配层

本文件只定义能力化适配策略，不维护具体模型品牌或型号清单。

## 1. 默认策略

- 所有工具先读取自己的短入口，再读取 `./.ai/runtime.md`。
- 详细规则按任务加载，不默认加载全部 `rules/`、`agents/`、`skills/`、`templates/`。
- 如果模型上下文足够，保持标准 Markdown 规则表达。
- 如果出现遗漏、混淆或上下文不足，按当前任务裁剪到必要规则。

## 2. 能力判断

| 维度 | 充足表现 | 受限时处理 |
|------|----------|------------|
| 上下文容量 | 能同时处理入口、当前规则、代码和验证输出 | 只保留 `runtime.md` 与当前任务规则 |
| 文件读取 | 能按路径读取 `.ai/` 和代码文件 | 要求人工粘贴必要规则或输出操作清单 |
| 工具能力 | 能稳定搜索、读取、测试和返回结果 | 先列命令、路径和风险，等待确认 |
| 指令遵循 | 能持续遵守红线和未验证口径 | 压缩成必须/禁止项逐条确认 |

## 3. 降级顺序

1. 保留平台短入口与 `runtime.md`。
2. 保留当前任务必读规则，移除无关端规则。
3. 保留当前任务涉及的代码、配置和验证输出。
4. Skills、templates、历史文档只保留被直接引用的内容。
5. 仍超限时，将规则压缩成“必须/禁止/交付要求”清单。

## 4. 固定不裁剪项

- 业务字典命名约束。
- OpenAPI `/v3/api-docs` 契约真源。
- `./.ai/api-status.yml` 接口状态真源。
- Mock 统一落点。
- 高风险操作确认。
- 未验证/未执行测试/未编译验证口径。

## 5. 交付说明

只有发生降级时才需要说明：实际加载文件、裁剪内容、降级原因和未验证范围。未降级时不需要记录模型型号。
