# 前后端契约一致性检查

## 概览

本 skill 以 OpenAPI 文档（`/v3/api-docs`）为唯一真源，逐项对照前端 API 调用代码，输出不一致项清单。

默认原则：

- OpenAPI 文档是契约真源，前端必须对齐
- 发现不一致时，不自动修改任何代码，只输出问题清单
- 所有结论基于静态阅读，不执行运行时验证

## 工作流

### 第 1 步：获取后端 OpenAPI 文档

默认路径（需后端服务运行）：

```text
http://localhost:48080/v3/api-docs
```

若后端未运行，可从以下替代来源获取：

- 项目内保存的 `api-docs.json` 快照
- knife4j 导出文档（`/v3/api-docs/swagger-config`）
- 人工提供的接口文档

### 第 2 步：提取前端 API 调用清单

在前端项目中检索所有 API 调用点：

```bash
rg -n "url:\s*['\"]" src/api/          # 接口路径定义
rg -n "export function" src/api/       # API 函数签名
```

重点关注：

- `{{dirs.frontend}}/src/api/` 下的所有 API 封装文件
- `{{dirs.frontend}}/src/views/` 中的直写请求（如有）

### 第 3 步：对照检查矩阵

对照以下维度逐接口检查：

| 检查维度 | 后端（OpenAPI） | 前端（API 调用） | 是否一致 |
|----------|----------------|------------------|----------|
| 请求路径 | `/edge/node/page` | 前端 URL 路径 | ✅/❌ |
| 请求方法 | GET/POST/PUT/DELETE | 前端 method | ✅/❌ |
| 请求参数 | 参数名、类型、是否必填 | 前端传参字段 | ✅/❌ |
| 响应结构 | 字段名、类型、嵌套结构 | 前端解构字段 | ✅/❌ |
| 分页参数 | pageNo/pageSize 命名 | 前端分页字段名 | ✅/❌ |
| 枚举值 | 枚举定义与可选值 | 前端映射表 | ✅/❌ |
| 错误码 | HTTP 状态码与业务错误码 | 前端错误处理分支 | ✅/❌ |

### 第 4 步：输出不一致清单

```markdown
## 契约一致性检查报告

**检查时间**：YYYY-MM-DD HH:mm
**契约来源**：http://localhost:48080/v3/api-docs

### 检查汇总

- 总接口数：N
- 一致：N
- 不一致：N
- 前端多余调用：N
- 后端未消费接口：N

### 不一致项详细清单

| 接口 | 问题类型 | 后端定义 | 前端实际 | 严重级别 | 建议修复 |
|------|----------|----------|----------|----------|----------|
| /edge/node/page | 字段缺失 | totalCount | total | P1 | 前端字段改为 totalCount |
| /edge/node/create | 参数名不一致 | nodeName | node_name | P2 | 统一为 nodeName |

### 验证边界

- 检查方式：静态代码对照
- 未验证项：运行时响应结构、复杂类型嵌套、文件上传
```

### 第 5 步：严重级别定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0 阻断** | 前端调用路径/方法与后端完全不一致 | URL 拼写错误、方法类型错误 |
| **P1 高风险** | 字段缺失或类型不匹配，会导致运行时崩溃 | 响应字段改名后前端未更新 |
| **P2 中风险** | 字段名不一致但类型兼容，可能静默失败 | 分页参数名不匹配 |
| **P3 低风险** | 多余字段、未消费字段、废弃接口残留 | 后端返回了前端不用的字段 |

## 输出要求

输出检查报告时，至少包含：

- 本次检查目标
- 检查样本范围
- 不一致项清单（含严重级别）
- 修复建议
- 验证边界（验证方式、未验证项与原因）

## 禁止事项

- 不要在不一致检查自动修改任何前端或后端代码
- 不要把"代码读了一遍"表述为"已通过验证"
- 不要假设运行时行为与静态代码一致
- 不要在缺少 OpenAPI 文档时用前端代码反推后端契约
