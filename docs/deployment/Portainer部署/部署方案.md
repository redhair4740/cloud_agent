# Portainer 部署方案

## 1. 文档目标

- 适用环境：单机 Docker、Docker Compose、Portainer Stack 管理的测试或轻量生产环境。
- 部署对象：`moon_cloud_backend/vmesh-server` 后端服务、`moon_cloud_frontend` 前端静态站点和 nginx。
- 当前状态：仓库内存在 Docker HOWTO 与 Jenkinsfile 示例，但旧文档中的 `WF_VMesh_Coud`、`WF_VMesh_Coud_UI` 路径已过时。
- 验证口径：本次只做静态整理，未执行 Jenkins、Harbor、Docker、Portainer 联调。

## 2. 架构与依赖

推荐发布链路：

```text
后端 Jenkins 构建 jar 和镜像
  -> 推送镜像仓库
  -> Portainer 重部署后端 Stack

前端 Jenkins 构建静态产物和 nginx 镜像
  -> 推送镜像仓库
  -> Portainer 重部署前端 Stack
```

边界：

- Jenkins 负责 build 和 push。
- 镜像仓库负责保存版本镜像。
- Portainer 负责 Stack 部署、重建和回滚。
- PostgreSQL、Redis、EMQX、对象存储等中间件默认作为外部依赖，不强行塞进业务 Stack。
- 前后端独立发布，避免一个端的小改动迫使另一个端重发。

## 3. 发布单元

| 发布单元 | 当前证据 | 说明 |
|----------|----------|------|
| 后端服务 | `moon_cloud_backend/vmesh-server`、`VMeshServerApplication` | Spring Boot 运行入口 |
| 后端构建 | `moon_cloud_backend/script/jenkins/Jenkinsfile` | 示例命令含 `mvn clean package -Dmaven.test.skip=true` |
| Docker Compose | `moon_cloud_backend/script/docker/Docker-HOWTO.md` | 示例命令含 `docker compose --env-file docker.env up -d` |
| 前端工程 | `moon_cloud_frontend/package.json` | 脚本含 `pnpm build:prod`、`pnpm build:local` |
| 原型工程 | `moon_cloud_design/package.json` | 独立原型构建，不作为业务发布主单元 |

## 4. 配置项

| 配置项 | 示例值 | 说明 | 来源 |
|--------|--------|------|------|
| 后端镜像名 | `<registry>/vmesh-server:<tag>` | 后端服务镜像 | 部署平台约定 |
| 前端镜像名 | `<registry>/moon-cloud-frontend:<tag>` | 前端 nginx 镜像 | 部署平台约定 |
| `SPRING_PROFILES_ACTIVE` | `prod` / `dev` | 后端运行环境 | Spring Boot 配置 |
| 数据库连接 | 使用环境变量或外部配置文件 | PostgreSQL 等外部依赖 | 部署环境 |
| Redis 连接 | 使用环境变量或外部配置文件 | 缓存外部依赖 | 部署环境 |
| `vmesh.edge.runtime.*` | 使用环境变量注入 | Edge MQTT/EMQX 配置 | `EdgeRuntimeProperties` |

敏感配置不写入文档和仓库，不在交付说明中回显明文。

## 5. 部署步骤

1. 前置检查：
   - 确认 PostgreSQL、Redis、EMQX 等外部依赖可访问。
   - 确认镜像仓库账号、Portainer Stack 权限、共享 Docker 网络。
   - 确认后端外部配置文件或环境变量已脱敏配置。
2. 后端构建与发布：
   - 在 `moon_cloud_backend` 执行 Maven 构建。
   - 仓库示例命令为 `mvn clean package -Dmaven.test.skip=true`。
   - 构建镜像并推送到镜像仓库。
3. 前端构建与发布：
   - 在 `moon_cloud_frontend` 执行 `pnpm i` 和 `pnpm build:prod`。
   - 将构建产物放入 nginx 镜像。
   - 推送镜像到镜像仓库。
4. Portainer 发布：
   - 后端 Stack 和前端 Stack 分开导入。
   - 更新镜像 tag 后重部署。
   - 后端服务健康检查通过后，再切前端。
5. 发布后验证：
   - 访问后端 `/admin-api` 相关接口。
   - 访问前端页面并验证登录、动态菜单、核心业务页面。
   - 查看后端日志、nginx 日志和 Portainer 容器状态。

## 6. 回滚

- 镜像回滚：Portainer Stack 切回上一版镜像 tag 后重部署。
- 配置回滚：恢复上一版环境变量或外部配置文件。
- 前端回滚：前端 Stack 单独回滚，不要求后端回滚。
- 后端回滚：后端 Stack 单独回滚，若涉及数据库结构变更，必须先确认 SQL 回滚方案。
