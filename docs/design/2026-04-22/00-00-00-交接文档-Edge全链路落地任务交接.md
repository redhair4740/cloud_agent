## 4. 当前明确风险

### 4.1 编译风险

本轮执行过离线编译探针：

```bash
cd /Users/yo/项目/ai_vision/cloud_agent/cloud
mvn -pl vmesh-module-edge -DskipTests -o -q compile
```

真实结果：失败。

失败原因不是已确认的源码语法错误，而是本地 Maven 依赖未缓存完整，缺少：

- `vmesh-spring-boot-starter-biz-tenant`
- `vmesh-spring-boot-starter-security`
- `vmesh-spring-boot-starter-mybatis`
- `vmesh-spring-boot-starter-redis`
- `vmesh-spring-boot-starter-excel`

结论：

- 当前不能写“已编译通过”
- 下一台机器应优先先把依赖下载完整，再做真实编译

### 4.2 清理风险

下一步建议做“安全清理”，但这一步本质上涉及删除重复文件，属于项目规则下的谨慎项。

建议清理对象：

- `runtime` 包下重复的 `EdgeNodeCredentialDO`
- `runtime` 包下重复的 `EdgeNodeRuntimeSnapshotDO`
- 未被主链使用的 `EdgeMonitorNodeDetailRespVO`

清理前必须先再次确认：

1. 当前主链 import 全部指向 `node` / `monitor` 包版本
2. 没有 XML mapper、反射、序列化或外部生成代码依赖这些重复类
3. 删除后再执行一次全模块编译
