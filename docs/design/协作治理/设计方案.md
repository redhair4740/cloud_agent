# 协作治理设计方案

## 1. 背景

- 需求来源：旧治理评审和三项目协作分析中有稳定结论，但 dated 文档本身属于历史快照。
- 当前现状：项目协作规范真源已收敛到 `.agents/`，包括 `rules/`、`skills/`、`references/`。
- 触发原因：需要把稳定治理口径整理成当前可阅读的项目文档，而不是继续保留多份旧评审和分析正文。

## 2. 目标

| 编号 | 目标 | 验收指标 |
|------|------|----------|
| G-1 | 明确规范真源 | 能定位 `.agents/rules`、`.agents/skills`、`.agents/references` 的职责 |
| G-2 | 明确三工程协作边界 | 原型、前端、后端各自职责清楚 |
| G-3 | 明确 API-First | 接口契约以 `/v3/api-docs` 为唯一真源 |
| G-4 | 明确文档治理 | 活文档、记录文档、索引与未验证口径一致 |

## 3. 规范资产边界

| 资产 | 路径 | 职责 |
|------|------|------|
| 稳定规则 | `.agents/rules/*.md` | 仓库基线、接口契约、后端、前端、文档、动态菜单等稳定约束 |
| 可执行流程 | `.agents/skills/*/SKILL.md` | 后端、前端、契约检查、文档模板、部署发布、review 等任务流程 |
| 事实参考 | `.agents/references/*.md` | 接口状态模型、业务词表等可增长事实 |
| 项目入口 | `AGENTS.md` | 当前工具可读的项目级基础行为 |
| 文档输出 | `docs/**` | 当前有效项目文档 |

治理原则：

1. Rule 只写稳定约束和边界，不写执行流程。
2. Skill 只写可复用流程，不复制通用红线全文。
3. Reference 只写可查阅事实，不承载任务调度。
4. 历史方案、临时分析、执行记录不写入 `.agents`。
5. 当前有效知识沉淀到 `docs`，历史记录不作为默认入口。

## 4. 三工程协作边界

| 工程 | 路径 | 角色 |
|------|------|------|
| 后端 | `moon_cloud_backend` | Spring Boot 服务、OpenAPI 生成源、权限、数据模型、业务逻辑 |
| 前端 | `moon_cloud_frontend` | 管理后台页面、动态路由、API SDK、交互和状态 |
| 原型 | `moon_cloud_design` | 页面结构、视觉和交互目标参考，不等同于接口契约真源 |

协作规则：

- 原型只定义体验目标，不能直接替代 OpenAPI、数据库字段或后端状态机。
- 前端不得猜字段名、枚举、分页结构和错误码。
- 后端接口变更必须反映到 Controller 注解和 `/v3/api-docs`。
- 动态菜单相关变更必须同步前端路由、后端菜单数据、按钮权限和页面可见性。

## 5. API-First 口径

接口契约真源：

```text
/v3/api-docs
```

执行顺序：

1. 后端先提供 Controller 骨架和完整 OpenAPI 注解。
2. 导出或提供 `/v3/api-docs`。
3. 前后端确认路径、方法、参数、字段、错误码、鉴权和分页语义。
4. 接口负责人维护接口状态。
5. 后端实现 Service。
6. 前端按 OpenAPI 实现类型、调用和页面行为。
7. 使用契约检查流程做一致性验证。

接口状态只描述联调可用性，不替代字段契约。状态枚举以 `.agents/references/interface-status-model.md` 为准，AI 不得自行把接口提升为 `ready`。

## 6. 文档治理口径

| 类型 | 路径规范 | 说明 |
|------|----------|------|
| 活文档 | `docs/<类型目录>/<主题短名>/<类型标签>.md` | 设计方案、API文档、部署方案 |
| 记录文档 | `docs/<类型目录>/yyyy-MM-dd/hh-mm-ss-类型标签-文档名称.md` | 交接、review、工单、治理评审等快照 |
| 索引 | `docs/README.md` | 当前有效主题入口 |

本次 `docs` 是一次重整输出：只保留当前有效主题，不迁移旧 `review-tracking` 和 dated 历史快照。旧治理评审的稳定结论已吸收进本文，旧正文不再作为当前入口。

## 7. 验收与风险

验收标准：

- [ ] 新增文档带 YAML frontmatter。
- [ ] 活文档包含 `last_updated`、`supersedes`、`related` 和变更日志。
- [ ] 关键判断包含文件路径、命令、配置键、接口路径或方法名。
- [ ] 未执行测试、编译、联调时明确写“未验证/未执行测试/未编译验证”。

风险：

- README 中存在旧技术栈口径，优先以 `pom.xml`、`package.json`、`.agents/rules` 为准。
- `moon_cloud_design/README.md` 是 GitLab 模板，不作为规范真源。
- `docs_old/README.md` 存在迁移后残留路径和状态滞后，不作为当前索引真源。

未验证项：

- 未执行构建和测试。
- 未读取实时 `/v3/api-docs`。
- 未做文档渲染工具检查。
