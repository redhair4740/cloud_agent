# 边缘节点状态语义设计方案

## 1. 背景

- 需求来源：统一 Edge 节点在台账、激活、注册、连接、运行五层状态中的语义，避免前端用单个状态字段承载所有含义。
- 当前现状：`EdgeNodeRespVO`、`EdgeNodeStatsRespVO`、`EdgeNodeRuntimeSnapshotDO`、`EdgeMonitor*RespVO` 已体现分层状态。
- 触发原因：旧文档把状态流转记录在 dated 文档中，当前需要作为活文档保留。

## 2. 目标

| 编号 | 目标 | 验收指标 |
|------|------|----------|
| G-1 | 分清台账开关和运行状态 | `enabledStatus` 不表示在线或运行健康 |
| G-2 | 分清激活和注册阶段 | `activationStatus` 表示当前凭证版本是否激活，`registrationStatus` 表示注册链路阶段 |
| G-3 | 分清连接态和运行态 | `connectionState` 表示 MQTT 在线，`runtimeStatus` 表示运行健康 |

## 3. 状态分层

| 层级 | 字段 | 存储/来源 | 语义 |
|------|------|-----------|------|
| 台账开关 | `enabledStatus` | `edge_node.enabled_status`、`EdgeNodeDO` | 是否允许接入链路，取值 `0/1` |
| 激活结果 | `activationStatus` | `edge_node.activation_status`、`EdgeNodeRespVO` | 当前 ACTIVE 凭证版本是否完成激活闭环，取值 `0/1` |
| 注册阶段 | `registrationStatus` | `edge_node_runtime_snapshot.registration_status` | `PREPARED`、`HANDSHAKING`、`ACTIVATED` |
| 连接态 | `connectionState` | `edge_node_runtime_snapshot.connection_state` | `UNKNOWN`、`ONLINE`、`OFFLINE` |
| 运行态 | `runtimeStatus` | `edge_node_runtime_snapshot.runtime_status` | `OFFLINE`、`ONLINE`、`RUNNING`、`WARNING` |
| 物理绑定 | `hardwareFingerprint` | `edge_node_credential.hardware_fingerprint` | 当前凭证版本绑定的物理节点指纹 |

## 4. 流转规则

1. 节点创建后默认允许进入待激活状态，真实默认值以 `EdgeNodeServiceImpl` 和 SQL 初始化为准。
2. `enabledStatus=0` 时，认证和 ingest 应拒绝。
3. `authenticate -> register` 成功后，当前凭证版本进入已激活。
4. 首次 `register` 必须携带 `hardwareFingerprint`，后续 `heartbeat`、`device.report`、`task_progress` 必须保持同一指纹。
5. 刷新凭证后旧 ACTIVE 凭证失效，节点回到待激活，运行快照收敛到离线。
6. `client.connected` 推进连接态；`client.disconnected` 回落连接态和运行态。
7. `heartbeat` 和 `task_progress` 只有在节点已激活且指纹匹配时才允许推进运行态。
8. 心跳超时由 `EdgeRuntimeServiceImpl#scanHeartbeatTimeouts` 巡检，默认延迟配置来自 `vmesh.edge.runtime.heartbeat-scan-interval-ms`。

## 5. API 映射

| 接口 | 字段/口径 |
|------|-----------|
| `GET /edge/node/get` | 输出 `enabledStatus`、`activationStatus`、`registrationStatus`、`connectionState`、`runtimeStatus` |
| `GET /edge/node/page` | 支持 `enabledStatus`、`activationStatus`、`registrationStatus`、`connectionState`、`runtimeStatus` 筛选 |
| `GET /edge/node/stats` | `pending` 按 `activationStatus=0`；`online/offline/running/warning` 由 snapshot 与 runtime 口径统计 |
| `POST /edge/runtime/ingest` | 对未启用、未激活、指纹缺失/不匹配、Topic 不匹配事件拒收 |
| `GET /edge/monitor/heartbeat/page` | 使用运行态快照展示心跳状态、槽位和成功率 |

## 6. 影响范围

- 后端：`EdgeNodeDO`、`EdgeNodeCredentialDO`、`EdgeNodeRuntimeSnapshotDO`、`EdgeRuntimeServiceImpl`、`EdgeMonitorServiceImpl`。
- 前端：`moon_cloud_frontend/src/api/edge/node.ts`、`monitor.ts`、`views/edge/management/**`、`views/edge/monitor/**`。
- 数据库：本文不新增 SQL；字段以当前 DO 和既有表结构为证据。

## 7. 风险与回滚

- 风险：若前端继续使用兼容字段 `status` 作为唯一状态，会混淆待激活、离线和告警。
- 风险：硬件指纹只能防止普通凭证复制误用，不能替代设备证书、TPM 或安全芯片。
- 回滚方案：实现层面保留旧兼容字段展示，但新增页面和接口不得把它作为唯一真源。
- 未验证项：未执行接口联调、未执行数据库迁移实跑、未执行编译测试。
