# 项目文档索引

## 使用约定

- 适用范围：设计方案、交接文档、治理评审、协作分析、部署方案和其他需要持续追踪的 Markdown 工件。
- 分类规则：文档分为活文档（`lifecycle: living`）和记录文档（`lifecycle: record`）。
- 活文档：`设计方案`、`API文档`、`部署方案`，按主题目录维护当前有效版本，更新原文件并追加变更日志。
- 记录文档：`交接文档`、`治理评审`、`协作分析`、`review清单`、`Bug修复工单`、`重构工单`，按时间线保留快照。
- 命名规范：活文档使用 `docs/<类型目录>/<主题短名>/<类型标签>.md`；记录文档使用 `docs/<类型目录>/yyyy-MM-dd/hh-mm-ss-类型标签-文档名称.md`。
- `review-tracking/` 目录例外，使用 `review清单-hh-mm-ss-文档名称-状态.md`。
- 新建记录文档必须使用创建时的实际本地时间生成 `hh-mm-ss`；除非用户明确指定时间，禁止默认使用 `00-00-00` 占位。
- 索引维护：新增或更新活文档后必须更新“主题索引”；新增记录文档后必须挂到对应主题的关联列表。
- 验证口径：未执行真实验证时明确写"未验证"，不要把静态判断写成"已完成"。

## 类型标签枚举

| 类型标签 | 适用目录 | 说明 |
|---------|---------|------|
| `API文档` | api/ | 接口契约文档，专注 HTTP API 定义与字段描述 |
| `设计方案` | design/ | 技术方案、架构设计、实施路线图 |
| `交接文档` | design/ | AI 续接文档，记录进度与下一步 |
| `治理评审` | governance/ | 规范评估、治理优化建议 |
| `协作分析` | governance/ | 协作模式分析、项目分析 |
| `部署方案` | deployment/ | 部署方案、运维方案、发布最佳实践 |
