# 项目文档索引

## 使用约定

- 适用范围：设计方案、交接文档、治理评审、协作分析、部署方案和其他需要持续追踪的 Markdown 工件。
- 分类规则：文档按类型归入 design / governance / deployment / review-tracking 目录。
- 日期子目录：每个类型目录下按 `yyyy-MM-dd` 创建日期子目录。
- 命名规范：`hh-mm-ss-类型标签-文档名称.md`，时间戳在前保证默认排序按时间排列；`review-tracking/` 目录例外，使用 `review清单-hh-mm-ss-文档名称-状态.md`。
- 交接文档就近原则：AI 交接文档与对应的设计方案放在同一类型目录的同一日期子目录下，且使用相同的时间戳（hh-mm-ss），方便通过文件名直接识别关联关系。
- 索引维护：新增文档后必须更新本文件索引。
- 验证口径：未执行真实验证时明确写"未验证"，不要把静态判断写成"已完成"。

## 类型标签枚举

| 类型标签 | 适用目录 | 说明 |
|---------|---------|------|
| `API文档` | api/ | 接口契约文档，专注 HTTP API 定义与字段描述 |
| `设计方案` | design/ | 技术方案、架构设计、实施路线图 |
| `交接文档` | design/ | AI 续接文档，记录进度与下一步 |
| `治理评审` | governance/ | 规范评估、治理优化建议 |
| `协作分析` | governance/ | 协作模式分析、项目分析 |
| `部署方案` | deployment/ | 部署方案、运维方案、发布最佳实践 |

## 目录结构

```
docs/
├── README.md                     ← 本文件（索引与约定）
├── api/                          ← API 文档（独立目录，专注接口契约）
│   └── yyyy-MM-dd/
│       └── hh-mm-ss-API文档-名称.md
├── design/                       ← 设计方案（含交接文档）
│   └── yyyy-MM-dd/
│       ├── hh-mm-ss-设计方案-名称.md
│       └── hh-mm-ss-交接文档-名称.md    ← 同一设计方案产出的交接文档使用相同时间戳
├── governance/                   ← 治理评审与协作分析
│   └── yyyy-MM-dd/
│       ├── hh-mm-ss-治理评审-名称.md
│       └── hh-mm-ss-协作分析-名称.md
├── deployment/                   ← 部署方案
│   └── yyyy-MM-dd/
│       └── hh-mm-ss-部署方案-名称.md
└── review-tracking/              ← 评审跟踪
    ├── README.md
    └── yyyy-MM-dd/
        └── review清单-hh-mm-ss-文档名称-状态.md
```
