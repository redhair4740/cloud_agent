# 边缘监控与任务流水接口文档

## 1. 文档说明

- 目标：依据最新 `EdgeHealthMonitor.vue` 与 `EdgeTaskRecord.vue` 原型，定义边缘监控与任务流水目标 API 契约。
- 原型来源：
  - `wf_vmesh_cloud_design/src/views/EdgeHealthMonitor.vue`
  - `wf_vmesh_cloud_design/src/views/EdgeTaskRecord.vue`
- 当前源码参考：
  - `WF_VMesh_Coud/vmesh-module-edge/src/main/java/com/wf/vmesh/module/edge/controller/admin/monitor/EdgeMonitorController.java`
  - `WF_VMesh_Coud/vmesh-module-edge/src/main/java/com/wf/vmesh/module/edge/controller/admin/task/EdgeTaskController.java`
  - `WF_VMesh_Coud_UI/src/api/edge/monitor.ts`
  - `WF_VMesh_Coud_UI/src/api/edge/task.ts`
- OpenAPI 来源：未读取实时 `/v3/api-docs`，本文件是基于源码与原型产出的目标契约文档。
- 接口状态：
  - `partial`：源码已有接口，但未按本轮原型完成字段和语义确认。
  - `planned`：本轮原型新增目标接口，当前源码未发现对应 Controller 方法。
  - `contracted`：源码已有接口，路径、请求 VO 与响应 VO 已按当前文档契约对齐。
  - `deprecated`：后端可保留，但最新页面不再消费。
- 统一响应壳：除特别说明外，后端返回 `CommonResult<T>`；分页返回 `CommonResult<PageResult<T>>`，其中 `PageResult<T>` 包含 `list`、`total`。

## 2. 边缘监控 API

### 2.1 获取监控概览

- 接口状态：`partial`
- 当前源码：`EdgeMonitorController#getOverview`
- 方法：`GET /edge/monitor/overview`
- 权限：`edge:monitor:query`
- 前端用途：顶部统计卡。

#### 响应字段

| 字段 | 类型 | 必填 | 说明 | 当前源码状态 |
|------|------|------|------|--------------|
| heartbeatNodeCount | Long | 是 | 心跳监控节点数 | 需新增，原型需要 |
| heartbeatHealthRate | BigDecimal | 是 | 24h 心跳健康度，百分比 | 已有 |
| abnormalDeviceCount | Long | 是 | 当前连接设备异常数 | 需新增，原型需要 |
| systemWarningCount | Long | 是 | 系统状态预警节点数 | 可由 `highLoadCount` 演进 |
| activeAlertCount | Long | 否 | 活动告警数 | 已有，最新原型未单独展示 |
| registeringCount | Long | 否 | 正在注册节点数 | 旧字段，最新页面不再展示 |

#### 示例响应

```json
{
  "code": 0,
  "data": {
    "heartbeatNodeCount": 4,
    "heartbeatHealthRate": 98.50,
    "abnormalDeviceCount": 4,
    "systemWarningCount": 4,
    "activeAlertCount": 2
  }
}
```

### 2.2 分页获取心跳节点

- 接口状态：`partial`
- 当前源码：`EdgeMonitorController#getHeartbeatPage`
- 方法：`GET /edge/monitor/heartbeat/page`
- 权限：`edge:monitor:query`
- 前端用途：边缘节点心跳监控块。

#### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

| pageSize | Integer | 是 | 每页条数 |
| keyword | String | 否 | 节点名称或设备标识关键词 |
| status | String | 否 | 状态：`ONLINE`、`WARNING`、`OFFLINE` |

#### 响应字段 `EdgeHeartbeatNodeVO`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| nodeId | Long | 是 | 边缘节点 ID |

| heartbeatSlots | List<Boolean> | 是 | 最近心跳槽位，前端用于脉冲图 |

## 4. 枚举定义

### 4.1 任务类型

| 枚举 | 说明 |
|------|------|
| `OTA` | OTA 底座升级 |

| `REBOOT` | 设备重启 |

### 4.2 任务状态

当前源码参考：`EdgeTaskStatusEnum`。

| 枚举 | 说明 | 是否终态 |
|------|------|----------|
| `CREATED` | 已创建 | 否 |
| `QUEUED` | 排队中 | 否 |
| `DISPATCHED` | 已下发 | 否 |
| `ACKNOWLEDGED` | 已确认 | 否 |
| `PROCESSING` | 执行中 | 否 |
| `SUCCESS` | 成功 | 是 |
| `FAILED` | 失败 | 是 |
