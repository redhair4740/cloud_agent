# AI_Vision 项目协作入口

本文件承载项目级基线，不承载领域规则、技能索引或任务调度。

## 基础行为

- 全程使用简体中文沟通、分析、注释与交付说明。
- 关键判断必须给出可复查证据：文件路径、方法名、配置键、命令、参数或真实行为。
- 不用空泛表述替代可执行结论；无法确认时明确说明缺少的证据。
- 未执行真实验证时，必须明确标注"未验证/未执行测试/未编译验证"。
- 只有存在真实验证证据时，才可使用"已修复""已通过""已完成"等结论。
- 禁止把静态分析、阅读代码、局部推断表述成已验证结论。
- 声称编译通过时，必须同时给出真实编译命令、执行范围与结果。

## 注释基线

- 注释必须使用简体中文。
- 注释解释意图、边界、取舍、失败路径与风险，不复述显而易见的代码字面含义。
- 复杂流程的详细注释位置和交付口径按对应后端、前端或文档 skill 执行。

## 工作方式

- 优先遵守当前工具已经加载的规则与技能。
- 不在本文件中维护 rule 列表和skill 列表的调度关系。
- 可复用流程沉淀为 skill，稳定约束沉淀为 rule。
- 若任务涉及接口契约、字段、错误码、鉴权或联调验收，先对齐契约和验收口径，再实现。
- 交付说明必须包含：目标、落地文件、影响范围、关键边界、验证结果和未验证项。
- 跨端未达成一致前，不做不可逆实现决策。

## 风险边界

- 高风险操作必须先说明操作类型、影响范围、风险和回滚方式，并获得明确确认。
- 高风险操作包括：删除文件或目录、批量移动/重命名、数据库结构变更、生产环境 API 调用、外部服务写入、权限或环境变更。
- 禁止默认执行危险 Git 操作，例如硬重置、覆盖式回退、强推覆盖历史。
- 发现敏感信息时只允许定位和脱敏说明，禁止回显明文。
